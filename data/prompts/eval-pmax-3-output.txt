## Outcome digest

**What it does**
Produces a SQL migration plus review note that adds foreign keys which may be empty and which remove child rows automatically when the parent is deleted, with every cascade deletion logged to an audit trail.

**Key inputs**
Existing migrations and model files under migrations/*.sql and models/*.py.

**Done when**
All new foreign keys allow empty values, deleting a parent removes its children, and each cascade writes an audit entry with parent id, child id, and timestamp.
