## Outcome digest

**What it does**
Produces a single SQL migration script that adds a UUID v4 theme_db_id primary key to theme_db, drops NOT NULL on content_id, and backfills fresh UUIDs for every existing row.

**Key inputs**
Current schema.sql at HEAD and the scripts/populate_*.py files under Y:/Projects/theme_db/.

**Done when**
The script applies cleanly on a production-data copy, every existing row has a non-null theme_db_id, and content_id values are preserved.

**Quick sample**

    ALTER TABLE theme_db ADD COLUMN theme_db_id UUID;
    UPDATE theme_db SET theme_db_id = gen_random_uuid() WHERE theme_db_id IS NULL;
    ALTER TABLE theme_db ALTER COLUMN theme_db_id SET NOT NULL;
    ALTER TABLE theme_db DROP CONSTRAINT theme_db_pkey;
    ALTER TABLE theme_db ADD PRIMARY KEY (theme_db_id);
    ALTER TABLE theme_db ALTER COLUMN content_id DROP NOT NULL;
